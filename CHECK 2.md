# CHECK.md – Визуальная проверка GUI после реализации Итерации 2

## 1. Меню «Файл»
- [ ] В меню присутствуют пункты:
  - **«Загрузить архив…»** – открывает диалог выбора ZIP-файла.
  - **«Новая сессия»** – очищает все загруженные данные.

## 2. Загрузка архивов
- [ ] При первом запуске список чатов пуст.
- [ ] **Загрузите первый архив** – в списке появляются чаты с префиксом `[имя_файла]`.
- [ ] **Загрузите второй архив** (с другими чатами) – новые чаты добавляются в список, старые остаются.
- [ ] Если во втором архиве есть чаты с ID, совпадающими с уже загруженными, они **не дублируются**; появляется сообщение «Все чаты из этого архива уже загружены».
- [ ] После загрузки список чатов автоматически обновляется.

## 3. Отображение источников
- [ ] Каждый чат в списке имеет формат:  
  `[имя_архива] Заголовок чата`  
  (например, `[my_archive.zip] Приветствие`).
- [ ] Для чатов, которые могли быть загружены через старый механизм, отображается префикс `[Imported]`.

## 4. Фильтрация чатов
- [ ] Введите текст в поле фильтра – список отфильтровывается по заголовкам чатов **с учётом всех источников**.
- [ ] При очистке фильтра возвращается полный список.
- [ ] Префикс источника сохраняется в отфильтрованных элементах.

## 5. Выбор чатов и отображение сообщений
- [ ] **Одиночный выбор** – дерево сообщений показывает все пары выбранного чата.
- [ ] **Множественный выбор** (Ctrl+клик или Shift+клик) – дерево показывает сообщения из всех выбранных чатов.
- [ ] При клике на сообщение в дереве в нижней панели отображаются текст запроса и ответа.
- [ ] Если выбор чатов снят, дерево и детали очищаются.

## 6. Поиск
- [ ] Введите поисковый запрос, выберите поле («Запрос», «Ответ» или «Название чата»), нажмите «Найти».
- [ ] Результаты поиска отображаются в дереве (только найденные пары).
- [ ] Счётчик результатов (например, «1 / 5») обновляется.
- [ ] Кнопки «<» и «>» перемещают по результатам, подсвечивая найденный фрагмент в тексте.
- [ ] Кнопка «Сбросить» возвращает дерево к полному списку сообщений выбранных чатов.
- [ ] Режим «Live» – поиск запускается при каждом нажатии клавиши в поле ввода.

## 7. Навигация по сообщениям
- [ ] Выберите один чат или несколько, затем выберите какое-либо сообщение.
- [ ] Кнопки «← Предыдущая» и «Следующая →» переключают между сообщениями в рамках **текущего набора** (все сообщения из выбранных чатов).
- [ ] Индикатор позиции в нижней панели (например, «Чат: Название | Сообщение 3 из 10») обновляется.
- [ ] Если выбран только один чат, навигация идёт по его сообщениям.
- [ ] Если выбрано несколько чатов, навигация идёт по объединённому списку сообщений.

## 8. Редактирование и сохранение
- [ ] Измените текст запроса или ответа в нижней панели.
- [ ] Нажмите «Сохранить изменения» – появляется сообщение об успешном сохранении.
- [ ] Пометка `modified` устанавливается для данной пары (визуально можно проверить, например, по изменению цвета в дереве).
- [ ] Если изменений нет, кнопка «Сохранить изменения» сообщает «Нет изменений для сохранения».

## 9. Новая сессия
- [ ] Выберите пункт меню «Файл» → «Новая сессия».
- [ ] Список чатов становится пустым.
- [ ] Дерево сообщений очищается.
- [ ] Панель деталей очищается.
- [ ] Поле поиска сбрасывается, счётчик результатов становится «0 / 0».
- [ ] Кнопки навигации отключаются (серые).

## 10. Сообщения и ошибки
- [ ] При попытке загрузить не ZIP-файл (или битый архив) появляется окно с ошибкой.
- [ ] При загрузке архива, все чаты которого уже есть, появляется информационное сообщение.
- [ ] При успешной загрузке нового архива появляется сообщение с количеством добавленных чатов.

## 11. Поведение окна
- [ ] Измените размер окна, переместите разделители панелей.
- [ ] Закройте и снова откройте приложение – размеры и положение должны восстановиться (если этот функционал уже был реализован).

## 12. Регрессионное тестирование (старые функции)
- [ ] Убедитесь, что поиск, навигация, редактирование работают так же, как и до изменений (при наличии только одного архива).
- [ ] Проверьте, что открытие архива через двойной клик (если была такая возможность) или через командную строку (если реализовано) не сломано.
