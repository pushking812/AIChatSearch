# Концепция AI Chat Search
Описание несовсем актуальное.

## 1. Введение
AI Chat Search — приложение для просмотра и поиска по экспортированным архивам чатов AI (пока только DeepSeek, ZIP-файлы с conversations.json). 
Программа позволяет загружать несколько архивов для параллельного поиска по ним.

## 2. Основные сценарии использования (паттерны взаимодействия)
Приложение ориентировано на анализ и редактирование экспортированных архивов чатов AI. 
Ниже описаны ключевые пользовательские сценарии, которые поддерживаются текущей реализацией.

### 2.1 Загрузка нескольких архивов
**Цель:** пользователь хочет одновременно работать с чатами из разных периодов или разных аккаунтов.
- Пользователь запускает приложение.
- Выбирает пункт меню «Файл → Загрузить архив...» и указывает ZIP-файл.
- Чаты из архива добавляются к уже загруженным (если их id не дублируются).
- В списке чатов каждый чат отображается с префиксом имени файла-источника, что позволяет визуально различать происхождение.
- Пользователь может выполнить загрузку любого количества архивов - в список чатов добавятся только уникальные чаты.

### 2.2 Фильтрация чатов по названию
**Цель:** быстро найти нужный чат среди множества.
- В верхней части панели «Чаты» расположено поле фильтра.
- При вводе текста список мгновенно обновляется, показывая только те чаты, в заголовке которых содержится введённая 
  подстрока (регистронезависимо).
- Фильтр применяется ко всем загруженным чатам, независимо от источника.

### 2.3 Просмотр сообщений выбранных чатов
**Цель:** изучить содержимое одного или нескольких чатов.
- Пользователь выделяет один или несколько чатов в списке (с помощью Ctrl/Shift или кнопок «Выбрать все»/«Снять выделение»).
- В центральной панели (дерево сообщений) отображаются все пары сообщений из выбранных чатов, сгруппированные по чатам.
- Каждая пара показывается в виде строки: индекс, начало запроса, начало ответа. Звёздочка `*` рядом с индексом означает, 
  что пара была отредактирована.

### 2.4 Детальный просмотр и редактирование сообщения
**Цель:** прочитать полный текст запроса и ответа, внести исправления.
- Пользователь кликает на интересующую пару в дереве сообщений.
- В нижней панели открываются два текстовых поля: «Запрос» и «Ответ» с полным содержимым.
- При необходимости текст можно редактировать прямо в полях.
- После изменений нужно нажать кнопку «Сохранить изменения» – тогда обновлённый текст фиксируется в объекте пары, флаг `modified` 
  устанавливается, и выполняется автосохранение сессии.
- В дереве сообщений у соответствующей пары появляется звёздочка.

### 2.5 Навигация по сообщениям внутри чата
**Цель:** последовательно просматривать все сообщения выбранного чата.
- В нижней панели доступны кнопки «← Предыдущая» и «Следующая →».
- Они позволяют перемещаться по парам текущего чата (того, который был выбран последним в дереве).
- Текущая позиция отображается в виде «Чат: Название | Сообщение N из M».

### 2.6 Поиск по содержимому сообщений
**Цель:** найти все сообщения, содержащие определённый текст.
- В правой верхней части окна расположена строка поиска.
- Пользователь вводит запрос, выбирает область поиска (Название чата, Запрос, Ответ) и нажимает «Найти».
- Результаты отображаются в дереве сообщений: показываются только те чаты и пары, в которых найдено совпадение. 
  В скобках рядом с названием чата указывается количество найденных сообщений и общее число совпадений.
- При включённом режиме «Live» поиск запускается автоматически при вводе текста.
- Найденные фрагменты можно перебирать кнопками «<» и «>» – соответствующий текст подсвечивается жёлтым в панели 
  деталей, а фокус перемещается к нему.

### 2.7 Управление сессией
**Цель:** сохранить текущее состояние (загруженные архивы, отредактированные тексты) между запусками программы.
- Приложение автоматически сохраняет сессию при каждом изменении (добавление архива, редактирование пары) и при закрытии окна.
- При следующем запуске все ранее загруженные архивы и внесённые правки восстанавливаются в точности.
- Для начала работы «с чистого листа» предусмотрен пункт меню «Файл → Новая сессия», который очищает все источники и сбрасывает интерфейс.

### 2.8 Восстановление геометрии окна
**Цель:** сохранить удобное расположение панелей между сеансами.
- При изменении размеров окна или перемещении разделителей (sash) их пропорции запоминаются в конфигурационном файле.
- При повторном запуске окно и панели восстанавливаются в том же виде.

## 3. Общая архитектура

Приложение построено по модели **Model-View-Controller (MVC)** с дополнительными сервисами:

- **Модель** (`model.py`) – определяет сущности: `Chat`, `MessagePair`, `DataSource`.
- **Контроллер** (`controller.py`) – центральный класс `ChatController`, управляющий источниками, фильтрацией, навигацией, поиском и сессиями.
- **Представление** (`gui_components/`) – набор виджетов и окон, построенных на `tkinter`.
- **Сервисы** (`services/`) – вспомогательные модули: загрузка архивов, поиск, управление сессиями.

Все компоненты слабо связаны; контроллер выступает посредником между моделью и GUI.

## 4. Модель данных (`model.py`)

### 4.1 `MessagePair`
- Хранит одну пару сообщений: текст запроса (`request_text`) и ответ (`response_text`).
- Содержит временные метки (`request_time`, `response_time`), идентификаторы узлов из исходного JSON.
- Флаг `modified` показывает, были ли изменения в текстах.

### 4.2 `Chat`
- Имеет уникальный `id`, заголовок, даты создания/обновления.
- Содержит список `MessagePair` (метод `get_pairs()`).
- Позволяет добавлять пары (`add_pair`).

### 4.3 `DataSource`
- Представляет один загруженный ZIP-архив (или импортированный набор чатов).
- Атрибуты: `file_path` (путь к архиву), `chats` (список чатов), `id` (имя файла для отображения).
- Позволяет группировать чаты по источнику и отслеживать происхождение.

## 5. Контроллер (`controller.py`)

### 5.1 Структура `ChatController`

**Атрибуты:**
- `sources: List[DataSource]` – все загруженные источники.
- `known_chat_ids: Set[str]` – множество идентификаторов уже загруженных чатов (для исключения дубликатов).
- `_chat_source_map: Dict[str, DataSource]` – отображение id чата → источник (для быстрого получения имени источника).
- `chats: List[Chat]` – объединённый список всех чатов (для обратной совместимости).
- `filtered_chats: List[Chat]` – отфильтрованный по заголовку список.
- `_current_filter_query: str` – текущий фильтр.
- `current_chat`, `current_chat_pairs`, `current_index_in_chat` – состояние навигации.
- Сервисы: `_search_service`, `_session_manager`.

**Основные методы:**
- `add_source(file_path)` – загружает новый архив, добавляет только новые чаты (проверка по `id`).
- `clear_all_sources()` – удаляет все источники и сбрасывает состояние.
- `filter_chats(query)` – фильтрует чаты по заголовку.
- `search(...)` – делегирует поиск сервису `SearchService`.
- `prev_pair()`, `next_pair()`, `get_nav_state()`, `get_position_info()` – навигация по сообщениям.
- `save_session()` / `load_session()` – сохранение/восстановление сессии через `SessionManager`.

### 5.2 Управление уникальностью чатов
При добавлении нового источника извлекаются все чаты из архива, но в `known_chat_ids` и в источник попадают только те, 
чей `id` ещё не встречался. Это гарантирует отсутствие дубликатов.

## 6. Сервисы `services/`
### 6.1 `archive_loader.py`
- Функция `load_from_zip(zip_path) -> List[Chat]` – читает ZIP, парсит `conversations.json`, восстанавливает структуру 
чатов с учётом фрагментов `REQUEST`, `THINK`, `RESPONSE`. Возвращает список объектов `Chat`.

### 6.2 `search_service.py`
- Класс `SearchService` – выполняет поиск по тексту запроса/ответа или по заголовку чата.
- `search(chat, query, field)` – возвращает список пар, удовлетворяющих условию.
- `search_with_positions(chat, query, field)` – возвращает позиции вхождений для подсветки.

### 6.3 `session_manager.py`
- Класс `SessionManager` – отвечает за сохранение/загрузку списка источников.
- Использует `pickle` для сериализации, но данные преобразуются в словарь с версией (`_CURRENT_VERSION = "1.0"`), 
  что обеспечивает обратную совместимость и возможность будущего перехода на JSON.
- Методы `save(sources)` и `load()`.

## 7. Графический интерфейс (`gui_components/`)
### 7.1 Главное окно (`application.py`)
- Класс `Application(tk.Tk)` – собирает все панели, управляет меню, обрабатывает события закрытия.
- Создаёт контроллер (`ChatController`), меню «Файл» (Загрузить архив, Новая сессия).
- Инициализирует панели: список чатов, дерево сообщений, детали сообщения.
- Восстанавливает геометрию окна через `WindowStateManager`.

### 7.2 Панель списка чатов (`chat_list.py`)
- Класс `ChatListPanel` – отображает чаты в виде дерева (`ttk.Treeview`) с группировкой по источникам.
- Поле фильтра (по заголовку) с мгновенным обновлением.
- Поддержка множественного выбора, кнопки «Выбрать все» / «Снять выделение».
- При изменении выбора вызывается callback, который обновляет дерево сообщений.

### 7.3 Панель дерева сообщений (`message_tree.py`)
- Класс `MessageTreePanel` – показывает пары сообщений для выбранных чатов (или результаты поиска) в виде дерева.
- Каждая пара отображается с индексом (и звёздочкой `*` при модификации) и превью текста.
- При выборе пары вызывается callback для отображения в детальной панели.

### 7.4 Панель деталей сообщения (`message_detail.py`)
- Класс `MessageDetailPanel` – два текстовых поля (`Text`) для редактирования запроса и ответа.
- Подсветка результатов поиска (тег `search_match`).
- Метка с информацией о текущей позиции (чат, номер сообщения).

### 7.5 Контроллер поиска (`search_controller.py`)
- Класс `SearchController` – управляет состоянием поиска (текущий запрос, результаты, навигация по результатам).
- Содержит список `results` (кортежи `(chat, pair, field, start, end)`) и текущий индекс.
- Методы `next()`, `prev()`, `go_to(index)` обновляют интерфейс через callback.

### 7.6 Менеджер состояния окна (`window_state.py`)
- Класс `WindowStateManager` – сохраняет и восстанавливает размер окна и пропорции PanedWindow.
- Использует JSON-файл в папке `.config`.

### 7.7 Константы (`constants.py`)
- Содержит настройки: минимальные размеры панелей, цвета, пути к файлам конфигурации.

## 8. Процесс загрузки нескольких архивов
1. Пользователь выбирает «Файл → Загрузить архив...».
2. `Application.add_archive()` вызывает `controller.add_source(file_path)`.
3. `load_from_zip` загружает все чаты из архива.
4. Контроллер фильтрует новые чаты по `id`, добавляет их в новый `DataSource` и обновляет `known_chat_ids`, `_chat_source_map`.
5. Вызывается `_rebuild_filtered_chats()` – объединяет все чаты и применяет текущий фильтр.
6. `ChatListPanel.update_list()` перестраивает дерево, группируя чаты по источникам.
7. Происходит автосохранение сессии (`controller.save_session()`).

## 9. Сохранение и загрузка сессии
- При запуске приложения `Application.__init__` вызывает `controller.load_session()`.
- `SessionManager.load()` читает файл `.config/session.pkl`, преобразует данные обратно в объекты `DataSource`, `Chat`, `MessagePair`.
- Контроллер восстанавливает `sources`, `known_chat_ids`, `_chat_source_map` и обновляет интерфейс.
- При любых изменениях (добавление архива, редактирование пары) вызывается `controller.save_session()`, что обеспечивает 
  постоянное сохранение состояния.

## 10. Поиск и навигация
- Поиск выполняется по выбранным чатам (поддерживается множественный выбор).
- `SearchController` получает результаты с позициями, обновляет дерево сообщений, отображая только найденные пары (группировка по чатам).
- Кнопки «<» и «>» перемещают по результатам, подсвечивая найденный фрагмент в текстовом поле.
- При выборе пары из дерева (или из результатов поиска) `Application._on_tree_selected` проверяет, есть ли эта пара среди результатов, 
  и если да – синхронизирует индекс поиска и подсветку.

## 11. Обработка изменений и модификация
- Редактирование текста происходит непосредственно в полях `MessageDetailPanel`.
- Кнопка «Сохранить изменения» сохраняет новые тексты в объект `MessagePair`, устанавливает флаг `modified` и автосохраняет сессию.
- В дереве сообщений индекс пары отображается со звёздочкой (`*`), если `modified == True`.

## 12. Дальнейшее развитие
Текущая архитектура позволяет легко добавить:
- Удаление источника через контекстное меню.
- Экспорт изменённых архивов.
- Ленивую загрузку больших чатов (сейчас все данные всегда в памяти).
- Полнотекстовый индекс для ускорения поиска.

## 13. Заключение
Приложение успешно реализует поддержку нескольких источников данных, сохраняет состояние между сессиями, предоставляет удобный поиск и навигацию. 
Код организован по принципу единственной ответственности, что облегчает поддержку и расширение.
